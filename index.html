What is Wakanda ?

Wakanda is a full stack javascript platform for creating web, hybrid and native applications. Wakanda is composed of two main products: Studio: IDE that helps you troubleshoot your environment, develop your apps and test them locally. : hosts and runs your Wakanda backend application during development and production. Wakanda Studio gives you a complete view of your application from the front to the back : Application Scaffolding: compose your app by selecting from a preconfigured list of templates : Backend: JavaScript, TypeScript.. Web: Angular4, Angular1.. (soon Aurelia and ReactJS) Mobile: Ionic2, Ionic1.. (soon React Native)

Wakanda Server

Wakanda Server is a multi-threaded server containing : JavaScript Application Server Uses V8 as JavaScript runtime Exposes a rich API to handle filesystem, sockets, HTTP Requests, built-in or remote databases, workers.. Provides a debugger based on Chrome DevTools to debug the different running JavaScript contexts (one context per thread) Wakanda HTTP Server Serves static HTTP files for your applications Used by other server components to expose Wakanda's REST API Can be used by your application's SSJS code to add custom HTTP patterns.. Data Abstraction Layer Handles the connection with data-sources (built-in WakandaDB or remote databases) Exposes a rich JavaScript API to access and manipualte data Provides events associated to data manipulation Secures access to data Exposes an automatically generated REST API to access and manipulate data WakandaDB (built-in dataBase)

Overview

Wakanda provides a way to run JavaScript code and system processes in separate threads : Three types of workers are available: SystemWorker, SharedWorker, NodeWorker A Worker proxy is an object that helps you communicate with the worker during its lifetime. Use a : System Worker to run an external system application (ex: html2pdf, shell, ...). Node Worker to start a Node JS thread that will run code in the background and that you can communicate with from anywhere in your app. Shared Worker to start a Wakanda JS thread that will run code in the background and that you can communicate with from anywhere in your app.
